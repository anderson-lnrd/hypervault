Getting Started

Guia completo para instalar, configurar e comecar a usar o HyperVault no seu projeto Angular.

Requisitos

Antes de comecar, certifique-se de ter:

  • obrigatorio Angular 21+
  • obrigatorio Node.js 20+
  • recomendado Angular CLI instalado globalmente

1. Instalacao (recomendado)

Use ng add para instalar e configurar tudo automaticamente — fontes, CSS variables, scrollbar e SCSS include paths:

ng add hypervault

O schematic faz automaticamente:

  • auto Adiciona fontes (Space Grotesk, Space Mono, Material Icons) ao index.html
  • auto Injeta CSS variables (design tokens) no styles.scss
  • auto Importa o scrollbar customizado
  • auto Configura stylePreprocessorOptions.includePaths no angular.json

Opcoes disponiveis:

# Sem scrollbar customizado
ng add hypervault --no-scrollbar

# Especificar projeto em monorepo
ng add hypervault --project=meu-app

1b. Instalacao Manual

Se preferir configurar manualmente, instale o pacote:

npm install hypervault
# ou
yarn add hypervault
# ou
pnpm add hypervault

E siga os passos 2, 3 e 4 abaixo.

2. Fontes manual

O HyperVault usa Space Grotesk (sans) e Space Mono (mono) para tipografia, e Material Icons para icones. Adicione ao seu index.html:

<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link href="https://fonts.googleapis.com/css2?family=Space+Grotesk:wght@400;500;600;700&family=Space+Mono:wght@400;700&display=swap" rel="stylesheet" />
<link href="https://fonts.googleapis.com/icon?family=Material+Icons" rel="stylesheet" />

Passo desnecessario se voce usou ng add hypervault.

3. Variaveis CSS manual

Defina os design tokens no seu styles.scss global. Estes sao os valores padrao do tema Cyber:

:root {
  /* Cores */
  --background: #0f0f0f;
  --foreground: #fafafa;
  --card: #1a1a1a;
  --muted: #2a2a2a;
  --muted-foreground: #a3a3a3;
  --input: #242424;
  --border: #fafafa;
  --primary: #39ff14;
  --primary-foreground: #0f0f0f;
  --secondary: #ff1493;
  --secondary-foreground: #0f0f0f;
  --accent: #ffff00;
  --accent-foreground: #0f0f0f;
  --destructive: #ff4444;
  --destructive-foreground: #fafafa;

  /* Tipografia */
  --font-sans: 'Space Grotesk', sans-serif;
  --font-mono: 'Space Mono', monospace;

  /* Neo-Brutalist */
  --border-width: 4px;
  --border-radius: 0px;
  --shadow-offset: 4px;
}

body {
  font-family: var(--font-sans);
  background-color: var(--background);
  color: var(--foreground);
  margin: 0;
}

Ou use o HyperThemeService para aplicar temas programaticamente — veja a pagina Theming.

4. Scrollbar (opcional) manual

Importe o modulo de scrollbar para customizar todas as barras de rolagem:

// styles.scss
@use 'hypervault/styles/scrollbar';

Configure o SCSS include path no angular.json:

"stylePreprocessorOptions": {
  "includePaths": ["node_modules"]
}

Para reverter ao scrollbar nativo em qualquer elemento:

<div class="hyper-scrollbar-native">...</div>

5. Usando Componentes

Importe componentes diretamente nos seus standalone components via subpath imports:

import { Component } from '@angular/core';
import { HyperButton } from 'hypervault/button';
import { HyperIcon } from 'hypervault/icon';

@Component({
  selector: 'app-meu-componente',
  imports: [HyperButton, HyperIcon],
  template: `
    <button hyper-raised-button color="primary">
      <hyper-icon>rocket_launch</hyper-icon>
      Lancar!
    </button>
  `,
})
export class MeuComponente {}

Cada componente vive em seu proprio entry point — o tree-shaking garante que apenas o codigo usado entre no bundle.

7. Defaults Globais (opcional)

Configure valores padrao para componentes com provideHyperDefaults() no app.config.ts:

import { provideHyperDefaults } from 'hypervault/core';

export const appConfig = {
  providers: [
    provideHyperDefaults({
      button: { color: 'primary', size: 'md' },
      checkbox: { color: 'primary' },
      switch: { color: 'primary' },
      toolbar: { color: 'primary', sticky: true },
    }),
  ],
};

8. Theming

O HyperVault inclui 27 temas built-in. Use o HyperThemeService para trocar em runtime:

import { HyperThemeService } from 'hypervault/theming';

export class AppComponent {
  private themeService = inject(HyperThemeService);

  ngOnInit() {
    this.themeService.setTheme('dracula');
    this.themeService.setDarkMode(true);
  }
}

Ou use o componente <hyper-theme-picker /> para um seletor visual. Veja todos os temas em Theming.

9. Densidade

Controle o espacamento de componentes via diretiva [hyperDensity] (escala de 0 a -4):

<!-- Todos os componentes filhos ficam mais compactos -->
<div [hyperDensity]="-2">
  <button hyper-raised-button>Compacto</button>
  <hyper-form-field label="Email">
    <input type="email" />
  </hyper-form-field>
</div>

Nota: Anchor Scrolling com Sidenav

O anchorScrolling: 'enabled' do Angular Router usa window.scrollTo(), que nao funciona quando o scroll container e um elemento interno (como o hyper-sidenav-content). Para navegar corretamente para fragmentos de URL (routerLink fragment="..."), implemente um listener manual de NavigationEnd no componente raiz: 

import { NavigationEnd, Router } from '@angular/router';
import { filter } from 'rxjs';

export class AppComponent implements OnInit {
  private router = inject(Router);

  ngOnInit() {
    this.router.events
      .pipe(filter(e => e instanceof NavigationEnd))
      .subscribe(() => {
        const fragment = this.router.parseUrl(this.router.url).fragment;
        if (!fragment) return;
        // Aguarda o componente lazy-loaded renderizar antes de rolar
        setTimeout(() => {
          document.getElementById(fragment)?.scrollIntoView({
            behavior: 'smooth',
            block: 'start',
          });
        }, 150);
      });
  }
}

O scrollIntoView() identifica corretamente o ancestral rolavel mais proximo (o wrapper do sidenav), e a propriedade CSS scroll-margin-top nos elementos alvo garante que o conteudo nao fique escondido atras da toolbar sticky: 

/* styles.scss — offset para toolbar sticky */
[id] {
  scroll-margin-top: 4rem;
}
