Page Summary

Sumário automático de página. Detecta todos os pontos de ancoragem, gera um índice navegável com suporte a hierarquia, ícones customizados e três modos de posicionamento.

Visão Geral

O hyper-page-summary escaneia o DOM em busca de headings com id e constrói um índice hierárquico clicável. A seção ativa é destacada automaticamente via IntersectionObserver — o que está visível na tela fica marcado em tempo real.

Suporte nativo a estruturas aninhadas: h3 dentro de h2 vira um sub-item, e assim por diante. Ícones Material customizáveis por id.

Uso Básico

Importe e coloque o componente onde quiser. Por padrão escaneia .demo-page:

import { HyperPageSummary } from 'hypervault/page-summary';

@Component({
  imports: [HyperPageSummary],
  template: `<hyper-page-summary />`,
})

Para um container diferente use o input container:

<hyper-page-summary container=".minha-pagina" />

Posicionamento

Três estratégias de posicionamento via input position:

inline (padrão)

Renderiza no fluxo normal do documento. Ideal para colocar no início ou fim da página:

<hyper-page-summary position="inline" />

sticky

Fica colado ao topo enquanto o usuário rola. Requer que o componente esteja numa coluna de grid com align-items: start — exatamente como nesta página de demo: 

.meu-layout {
  display: grid;
  grid-template-columns: 1fr 220px;
  align-items: start;
}

<div class="meu-layout">
  <div class="conteudo">...</div>
  <hyper-page-summary position="sticky" />
</div>

fixed-end

Flutua sobre o conteúdo, fixo na borda direita da tela. Bom para páginas com layout de coluna única larga:

<hyper-page-summary position="fixed-end" />

Hierarquia (h2 → h3 → h4)

O componente constrói automaticamente uma árvore a partir dos níveis dos headings. h3[id] vira filho do h2[id] anterior, igual à estrutura de um PDF. Esta página usa exatamente isso — veja os sub-itens de "Posicionamento" no sumário ao lado.

Seletores customizados

Controle quais headings são capturados com o input selectors:

<!-- Apenas h2 e h3 -->
<hyper-page-summary selectors="h2[id], h3[id]" />

<!-- Qualquer elemento com id e classe específica -->
<hyper-page-summary selectors=".section-heading[id]" />

Ícones por Seção

Passe um objeto Record<id, iconName> para o input icons. Cada entrada mapeia o id do heading a um nome de ícone do Material Icons. Seções sem mapeamento usam ícones padrão por nível (tag / chevron_right / fiber_manual_record). 

@Component({
  template: `
    <hyper-page-summary
      [icons]="tocIcons" />
  `
})
export class MinhaPagina {
  tocIcons = {
    'instalacao':  'rocket_launch',
    'componentes': 'widgets',
    'theming':     'color_lens',
    'density':     'density_small',
  };
}

O sumário nesta página usa ícones customizados — veja ao lado.

Título do Painel

Configure o texto do cabeçalho com o input title. Passe uma string vazia para ocultar:

<hyper-page-summary title="Nesta página" />
<hyper-page-summary title="Sumário" />
<hyper-page-summary title="" /> <!-- sem cabeçalho -->

API Reference

Inputs

Input Tipo Padrão Descrição
container string .demo-page Seletor CSS do container a escanear
title string Nesta página Texto do cabeçalho (vazio = sem cabeçalho)
selectors string h2[id], h3[id], h4[id] Seletor CSS dos headings a capturar
icons Record<string,string> {} Mapa de id → nome de ícone Material
position HyperPageSummaryPosition inline inline | sticky | fixed-end

HyperTocEntry

interface HyperTocEntry {
  id: string;
  text: string;
  level: number;        // 2 | 3 | 4
  icon?: string;        // Material Icon name
  children: HyperTocEntry[];
}
